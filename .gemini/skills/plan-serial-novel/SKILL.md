# 連載小說企劃

## 工作流程

1. 讀取作品現有 `README.md` 與 `novel-bible/`；有既有設定時以其為準。
2. 釐清題材、受眾、篇幅與平台。資訊不足時使用適合 YouTube 有聲連載的繁體中文爽文預設。
3. 先建立故事承諾與限制，再設計角色、衝突、成長及章節。
4. 將結果寫入對應的 Bible 檔案，不把企劃資料混入正文。

每部作品直接建立於專案根目錄的 `作品名稱/`。正式有聲書稿只存於 `chapters/`，一章一檔，例如 `ch001.md`、`ch002.md`；不建立完整合併稿、閱讀版或其他文字複本。

## 必要企劃項目

- 作品名稱、題材與一句話賣點
- 核心故事承諾與三至四種主要爽點
- 世界規則、勢力、權力結構與隱藏真相
- 主角的慾望、弱點、被低估原因、能力與成長方向
- 能力或系統的觸發、限制、代價與升級方式
- 配角各自的目標、立場與關係變化
- 具有資源、動機與壓迫感的反派
- 分卷主線、時間線、伏筆及前十章大綱

## 章節大綱要求

每章列出：

```markdown
## 第 XXX 章：章名
- 章節目標：
- 主要衝突：
- 本章爽點或情緒回報：
- 角色或資源變化：
- 伏筆推進或回收：
- 章末懸念：
```

每章都必須推進主線、角色、資源或資訊，避免只有設定說明。小成長可每一至二章出現，中型轉折約每五至八章一次，每卷完成一次重大狀態改變。

## 原創與安全

不要直接模仿特定在世作者，不使用受版權保護角色作為主要角色，也不照搬既有作品設定。可以借鑑類型元素，但必須重新組合。避免性暴力、未成年情色、仇恨煽動、犯罪教學與危險行為操作指引。
