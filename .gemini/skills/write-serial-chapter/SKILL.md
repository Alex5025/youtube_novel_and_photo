# 連載章節寫作

## 寫作前

先讀取作品 Bible、角色設定、時間線、前一章與本章大綱。不得擅自改變姓名、能力限制、資源數量、角色關係或事件順序。發現矛盾時先採最接近既有文本的解法並標記問題。

## 預設規格

- 使用 UTF-8 繁體中文。
- 一般章節 2500 至 4000 個中文字；短內容 600 至 1200 字；長影片 6000 至 10000 字。
- 使用白話、直接、具畫面感且適合朗讀的短句與短段落。
- 拆開過長句子，以逗號、句號與換行控制自然停頓。
- 減少括號、表格、Emoji、特殊符號及容易造成 TTS 誤讀的縮寫。
- 對話人物必須可由上下文辨識，數字、英文與專有名詞讀法保持一致。
- 每次只建立一個 `作品名稱/chapters/chXXX.md`，且該檔只能包含一章。
- 章節檔就是正式有聲書稿，不另建閱讀版、完整合併稿或 TTS 複本。
- 不混入章節分析、生圖提示詞、YouTube 資料或其他不應被朗讀的製作註記。

## 節奏

1. 前 300 字內出現危機、選擇、衝突、身份反差或其他鉤子。
2. 每 600 至 900 字加入新的壓力、資訊、選擇或反轉。
3. 主角不能長期被動；忍耐或低調必須服務於明確策略。
4. 至少安排一次可辨識的情緒回報，例如翻盤、智取、升級、資源增加或立場改變。
5. 結尾引出新威脅、秘密、任務或更大的代價，不平淡收束。

## 角色規則

- 主角要有慾望、弱點、受壓原因與實質成長。
- 反派要有合理動機、資源與壓迫感，不靠無腦降智推進。
- 配角要有自己的目標與立場，不只是襯托或無條件崇拜主角。
- 對話需符合身份、情緒與當下資訊，不用旁白重複解釋。

## 輸出格式

檔案路徑：`作品名稱/chapters/chXXX.md`

```markdown
# 第 XXX 章：章名

可直接朗讀的章節內容。
```

不要在正式章節內加入「本章定位」「正文」「品質檢查」等製作標題。完成後自行檢查設定連續性、開頭鉤子、中段推進、情緒回報、章末懸念及朗讀流暢度。
