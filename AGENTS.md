# Repository Guidelines

## 專案目的

本專案用於製作可連載、可朗讀、可視覺化並可發布至 YouTube 的繁體中文有聲書。產出包含作品設定、可直接朗讀的分章稿、生圖提示詞、影片包裝與相關素材。

## 基本責任

- 先閱讀作品目錄內的 `README.md` 與 `novel-bible/`，再修改故事內容。
- 優先產出可直接使用的成品；資訊不足時採合理預設，並標示可調整項目。
- `chapters/chXXX.md` 是唯一正式文字稿，必須可直接用於 TTS 或錄音。
- 一個章節檔只能包含一章；生圖提示詞與 YouTube 資料另行按章保存。
- 不擅自覆寫既有設定、角色關係、時間線或使用者修改。
- 涉及求生、急救、法律或其他專業資訊時，發布前需查證可靠來源。

## 專案粗架構

```text
作品名稱/
  README.md
  novel-bible/       # 世界觀、角色、能力、劇情與時間線
  chapters/
    ch001.md         # 第一章可直接朗讀的正式稿
    ch002.md         # 第二章可直接朗讀的正式稿
  image-prompts/     # 每章生圖提示詞
  youtube/           # 每章標題、簡介與縮圖文字
  assets/            # 圖片、音訊與影片素材
```

每部作品直接使用專案根目錄下的獨立 `作品名稱/`。`chapters/chXXX.md` 是唯一文字來源，不建立閱讀版、完整合併稿或其他內容重複的文字稿。

## 編碼與檔名

- 所有文字檔使用 UTF-8 編碼，預設語言為繁體中文。
- Markdown 使用清楚標題、短段落及有語言標記的程式碼區塊。
- 製作子目錄使用小寫 kebab-case；作品目錄可使用正式作品名稱。
- 章號使用三位數，例如 `ch001.md`；每個檔案只能有一個章節。
- 不提交密碼、Token、API Key、Cookie、私鑰或 `.env`。
- 大型音訊、影片與生成圖片使用 Git LFS 或外部儲存，不直接放入一般 Git 歷史。

## 任務技能

依任務載入對應專案技能：作品企劃、章節寫作、生圖提示詞、YouTube 包裝、連載品質檢查或 Git 操作。技能位於 `.codex/skills/`，Gemini CLI 對應版本位於 `.gemini/skills/`。
